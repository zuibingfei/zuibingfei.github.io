Github Page+Hexo NexT+Gitment个人博客

发表于 2018-07-03 | 更新于: 2018-07-04 | 分类于 技术开发 | 阅读数

作为一个程序猿，那么就要有一个自己的博客。Github Page+Hexo是自己比较喜欢的，自己想怎么玩就怎么玩。GithubPage和Hexo是什么，自己就不多介绍了，一搜一大堆。这里说流程和遇到的问题。

安装前提

1.Node.js：生成Hexo静态页面，下载安装 Node.js
2.Git：本地Hexo提交到Github，下载安装 Git
这里说一下我的系统是 macOS High Sierra 10.13.5

安装Hexo 官方文档

上面的环境没问题，使用npm安装hexo很简单，打开终端执行命令，记得加 sudo

$ sudo npm install -g hexo

创建Hexo存储文件夹（如blog，以下都会用blog表示），初始化Hexo

$ cd blog
$ hexo init

同blog路径，初始化npm

$ npm install

Hexo命名执行

执行generate命令生成静态文件
$ hexo g
部署到本地服务器
$ hexo s

生成博客本地地址 http://localhost:4000 ，打开浏览器，输入地址，biu这就是你的博客了。

创建Github仓库

登录自己github账号，在个人中心Settings/Repositories，create a new Repository。Repository name格式必须是 username.github.io，username是你的github的username，仓库就OK了。 https://username.github.io 以后就是你的博客域名。

部署Hexo

blog目录下，打开 _config.yml文件，修改下面的配置，repository设置自己的仓库

Hexo命令署到github仓库，注意 ，必须执行hexo clean清除缓存

清除缓存
$ hexo clean
部署到本地服务器
$ hexo d

命令执行成功后，浏览器打开自己的博客 https://username.github.io，现在就大功告成了，自己可以大笑三声了。
这里先不讲绑定自己的服务器，以后绑定后更新

写作

你可以执行下列命令来创建一篇新文章。Hexo 有三种默认布局：post、page 和 draft，它们分别对应不同的路径。

$ hexo new [layout] <title>
布局 路径
post source/_posts
page source
draft source/_drafts

Hexo 使用 Markdown（或其他渲染引擎）解析文章。

NexT主题 官方文档

NexT主题是自己比较的风格，这里为大家推介一下，也可以到 Theme 找自己喜欢的主题。
命令行下载主题

$ cd blog
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

blog目录下，打开 _config.yml文件，找到theme字段，并将其值更改为next。
Hexo命令，注意 部署到本地，同样必须执行清除缓存，浏览器访问 http://localhost:4000

$ hexo clean
$ hexo g
$ hexo s

访问统计

打开blog/themes/next/_config.yml文件添加汉字就可以，safari浏览器uv统计不准确

其他设置根据官方文档，设置自己喜欢的样式就好了。

Gitment评论

注意 gitment的文档很久没有更新了，使自己走了不少弯路

  1. 注册 OAuth Application
    点击此处 来注册一个新的 OAuth Application。其他内容可以随意填写，但要确保填入正确的 callback URL（自己博客的域名，如username.github.io）。

  2. 引入 Gitment
    直接blog/themes/next/_config.yml修改，不要跟着文档转圈圈哦，上代码，github_repo一定写博客的域名 client_id注册申请的id，client_secret申请的secret

  3. 初始化评论
    页面发布后，你需要访问页面并使用你的 GitHub 账号登录（请确保你的账号是第二步所填 repo 的 owner），点击初始化按钮。不点击初始化，显示评论未开启
